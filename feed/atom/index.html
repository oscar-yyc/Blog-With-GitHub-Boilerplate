<code>socks5</code> 代理，二选一即可</p> <pre><code># http代理 $ git config --global http.proxy http://server:port $ git config --global https.proxy http://server:port # socks5代理 $ git config --global http.proxy socks5://127.0.0.1:10808 $ git config --global https.proxy socks5://127.0.0.1:10808</code></pre> <p>以 v2raryN 代理工具为例：</p> <pre><code># http代理 $ git config --global http.proxy http://127.0.0.1:10809 $ git config --global https.proxy http://127.0.0.1:10809 # socks5代理 $ git config --global http.proxy socks5://127.0.0.1:10808 $ git config --global https.proxy socks5://127.0.0.1:10808</code></pre> <p>取消 Git 代理：</p> <pre><code>$ git config --global --unset http.proxy $ git config --global --unset https.proxy</code></pre> <h2>配置NPM代理</h2> <p>NPM 原生支持 <code>http</code> 代理类型，但是不支持 <code>socks5</code> 代理类型，如果还想要使用 <code>socks5</code> 代理，可能还需要使用一个工具使用 <code>http</code> 监听 <code>socks5</code> 代理 （禁止套娃），此处不做讨论</p> <pre><code># http代理 $ npm config set proxy http://server:port $ npm config set https-proxy http://server:port</code></pre> <p>以 v2raryN 代理工具为例：</p> <pre><code>$ npm config set proxy http://127.0.0.1:10809 $ npm config set https-proxy http://127.0.0.1:10809</code></pre> <p>取消 NPM 代理：</p> <pre><code>$ npm config delete proxy $ npm config delete https-proxy</code></pre> <h2>配置Yarn代理</h2> <p>Yarn 原生就拥有比 NPM 更快的速度，类似 NPM，使用 <code>http</code> 代理（必要的话），打开 Terminal/Bash：</p> <pre><code># http代理 $ yarn config set proxy http://server:port $ yarn confit set https-proxy http://server:port</code></pre> <p>以 v2raryN 代理工具为例：</p> <pre><code>$ yarn config set proxy http://127.0.0.1:10809 $ yarn confit set https-proxy http://127.0.0.1:10809</code></pre> <p>取消 Yarn 代理：</p> <pre><code>$ yarn config delete proxy $ yarn config
